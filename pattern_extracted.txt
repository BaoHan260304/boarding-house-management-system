Assignment: Design Pattern Analysis and Application in Software Projects
Mục tiêu
Sau assignment này sinh viên phải:
Phân biệt được các nhóm Design Pattern. 
Nhận diện các vấn đề thiết kế trong hệ thống. 
Giải thích được khi nào nên hoặc không nên dùng pattern. 
Áp dụng pattern vào chính sản phẩm của nhóm. 
Minh họa bằng UML và code thực tế. 
Yêu cầu chung cho tất cả các nhóm
Mỗi nhóm chuẩn bị:
Phần 1. Tổng quan Pattern (10%)
Trình bày ngắn gọn:
Design Pattern là gì? 
Mục đích của Pattern. 
Các thành phần: 
Intent 
Problem 
Solution 
Consequences 
Phần 2. Phân tích và phân biệt Pattern (40%)
Với mỗi pattern được giao:
1. Design Problem
Trả lời:
Vấn đề thiết kế nào dẫn đến sự ra đời của pattern? 
Nếu không dùng pattern thì hệ thống gặp khó khăn gì? 
2. Cấu trúc Pattern
Trình bày:
Class Diagram 
Sequence Diagram

Ví dụ:

Phần 3.
 Á
p dụng vào sản phẩm nhóm (30%)
Nhóm phải lựa chọn:
Tối thiểu 2 pattern
để áp dụng vào sản phẩm đồ án của nhóm.
Mỗi pattern cần trình bày:
Hiện trạng
Thiết kế ban đầu:
Class nào đang có vấn đề? 
Vấn đề mở rộng? 
Vấn đề coupling? 
Thiết kế sau khi áp dụng Pattern
Trình bày:
Before Design 
After Design 
Bằng UML.
Lợi ích đạt được
Ví dụ:
Giảm coupling. 
Tăng cohesion. 
Hỗ trợ Open-Closed Principle. 
Dễ bảo trì. 
Phần 4. Demo Code (20%)
Mỗi pattern cần có:
Demo độc lập
Ví dụ Java:
public interface PaymentStrategy {
    void pay(double amount);
}
Demo trong sản phẩm nhóm
Ví dụ:
Payment Strategy trong hệ thống bán hàng. 
Observer trong notification. 
Builder trong report generation. 
Command trong workflow. 
Code phải giải thích:
Vai trò từng class. 
Luồng thực thi.
