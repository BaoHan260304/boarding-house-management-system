### UC 05: View Utility Bills
**Functional Description Template**
**UC ID and Name:** UC 05 - View Utility Bills
**Created By:** quannm (Nguyen Manh Quan)
**Date Created:** 15/07/2026
**Primary Actor:** Tenant
**Secondary Actors:** None
**Trigger:** Tenant wishes to check their monthly living expenses (rent, water, electricity) or review past invoices.
**Description:** Allows tenants to view detailed itemized invoices (rent, electricity, water) for the current month and historical data of previous months. (Maps to feature FE-05).
**Preconditions:** The Tenant is successfully logged into the system.
**Postconditions:** The Tenant successfully views the requested invoice details.
**Normal Flow:**
1. Tenant navigates to the "Billing" or "My Invoices" module.
2. The system displays a summary list of all invoices (Current and Historical) grouped by month.
3. Tenant clicks on a specific invoice for the current month.
4. The system retrieves and displays the detailed breakdown: Room Rent, Electricity (Previous vs Current Index, Total), Water, Service Fees, Total Amount, and Due Date.
5. The system displays the current status of the invoice (e.g., "Unpaid", "Paid", "Overdue").
**Alternative Flows:**
2a. Filter by Year: Tenant selects a specific year to view older historical invoices. The system updates the list accordingly.
**Exceptions:**
2a. No Invoices Found: If the tenant just moved in and no bills have been generated, the system displays "No invoices available at this time."
**Priority:** High / Must Have
**Frequency of Use:** Medium (Usually 1-3 times per month per tenant).
**Business Rules:** Tenants can only view invoices linked to the room they are currently or historically renting.
**Other Information:** None
**Assumptions:** Billing data is correctly generated by the Staff/System beforehand.

### UC 06: Pay Rent Online
**Functional Description Template**
**UC ID and Name:** UC 06 - Pay Rent Online
**Created By:** quannm (Nguyen Manh Quan)
**Date Created:** 15/07/2026
**Primary Actor:** Tenant
**Secondary Actors:** Bank / E-Wallet (System Actor)
**Trigger:** Tenant decides to settle an "Unpaid" utility bill/invoice via an online payment method.
**Description:** Executes invoice payments by connecting to the integrated Bank or E-Wallet gateways, allowing tenants to clear their monthly debts without cash. (Maps to feature FE-06).
**Preconditions:** The Tenant is logged in, and there is at least one "Unpaid" or "Overdue" invoice in their account.
**Postconditions:** The invoice status is updated to "Paid", and the system records the payment transaction.
**Normal Flow:**
1. Tenant views the details of an "Unpaid" invoice (Continuation of UC 05).
2. Tenant clicks the "Pay Now" button.
3. The system displays available payment methods (e.g., Bank Transfer, E-Wallet / MoMo).
4. Tenant selects a preferred payment method and confirms.
5. The system generates a payment request with a specific transaction amount and unique reference code (or QR code) and redirects to the payment gateway.
6. The Secondary Actor (Bank / E-Wallet) processes the transaction.
7. The system receives a successful transaction callback from the gateway.
8. The system updates the invoice status to "Paid", creates a digital receipt, and displays a "Payment Successful" message.
**Alternative Flows:**
5a. Manual Bank Transfer: Instead of an API redirect, the system displays the Landlord's bank details and a strict transfer syntax. Tenant transfers manually and clicks "I have paid". The system marks the bill as "Pending Verification" until Staff confirms.
**Exceptions:**
6a. Payment Failed/Cancelled: The Tenant cancels the transaction or the account has insufficient funds. The gateway returns a failure status. The system alerts "Payment Failed" and keeps the invoice as "Unpaid".
**Priority:** High / Must Have
**Frequency of Use:** Medium (Once a month per tenant).
**Business Rules:** Partial payments are not allowed; the invoice must be paid in full.
**Other Information:** Uses Strategy Pattern in code to handle different payment gateways easily.
**Assumptions:** The integrated Bank/E-Wallet APIs are highly available and responsive.

### UC 07: Submit Maintenance Requests
**Functional Description Template**
**UC ID and Name:** UC 07 - Submit Maintenance Requests
**Created By:** quannm (Nguyen Manh Quan)
**Date Created:** 15/07/2026
**Primary Actor:** Tenant
**Secondary Actors:** Staff (Receives the request)
**Trigger:** A tenant discovers a facility issue (e.g., broken pipe, electrical fault) and needs repair assistance.
**Description:** Sends notifications regarding damaged facilities (electricity, water, infrastructure) to the management team and allows the tenant to track the repair status. (Maps to feature FE-07).
**Preconditions:** The Tenant is logged in and currently holds an active rental contract.
**Postconditions:** A new maintenance request is recorded in the system with the status "Pending".
**Normal Flow:**
1. Tenant navigates to the "Maintenance" or "Report Issue" module.
2. Tenant clicks the "New Request" button.
3. The system displays a submission form.
4. Tenant enters a description of the issue, selects the category (e.g., Electrical, Plumbing), and optionally uploads photos of the damage.
5. Tenant clicks "Submit".
6. The system validates the inputs and saves the request to the database with a "Pending" status.
7. The system notifies the assigned Staff/Landlord and displays a "Request submitted successfully" confirmation to the Tenant.
**Alternative Flows:**
None.
**Exceptions:**
4a. Missing Description: Tenant leaves the description blank. The system blocks submission and prompts "Please describe the issue."
**Priority:** Medium / Should Have
**Frequency of Use:** Low (Occurs randomly when infrastructure fails).
**Business Rules:** Tenants can view the status of their own requests (Pending, In Progress, Completed) but cannot change the status.
**Other Information:** This UC is directly linked to UC 17 (Update Maintenance Status) handled by Staff.
**Assumptions:** None.

### UC 08: Input Billing Information
**Functional Description Template**
**UC ID and Name:** UC 08 - Input Billing Information
**Created By:** quannm (Nguyen Manh Quan)
**Date Created:** 15/07/2026
**Primary Actor:** Staff
**Secondary Actors:** None
**Trigger:** It is the end of the month, and Staff needs to record utility usage to generate invoices.
**Description:** Manages the manual data entry of monthly electricity and water consumption metrics for each room. The system uses these metrics to automatically calculate the final bill. (Maps to feature FE-05).
**Preconditions:** Staff is logged in with appropriate permissions. The rooms must have active tenants.
**Postconditions:** Monthly utility consumption is recorded, and an "Unpaid" invoice is generated for the room.
**Normal Flow:**
1. Staff navigates to the "Billing Management" module.
2. The system displays a list of active rooms that require billing for the current month.
3. Staff selects a room to input data.
4. The system retrieves and displays the previous month's electricity and water meter indices.
5. Staff inputs the current month's electricity and water meter indices.
6. Staff clicks "Generate Bill".
7. The system validates that the current indices are strictly greater than or equal to the previous indices.
8. The system calculates the total usage, multiplies it by the configured unit prices, adds the base room rent, and generates a final Invoice.
9. The system saves the Invoice with status "Unpaid" and displays a success message.
**Alternative Flows:**
None.
**Exceptions:**
7a. Invalid Meter Index: Staff enters a current index lower than the previous month's index. The system blocks the action and displays "Current index cannot be lower than previous index."
**Priority:** High / Must Have
**Frequency of Use:** Medium (Done once per room, per month).
**Business Rules:** Bills cannot be generated twice for the same room in the same month.
**Other Information:** None.
**Assumptions:** The unit prices for electricity and water are properly configured in System Settings (UC 19) beforehand.
